# 일본 부동산 경매 시장 분석

> 본 문서는 한국 법원경매 보조 도구(`auction-seven-omega`)를 일본 시장으로 확장할 가능성을 검토하기 위한 리서치 노트입니다.

## 결론부터

**일본에도 한국과 거의 동일한 구조의 법원 경매(不動産競売, 후도산쿄바이) 제도가 존재합니다.** 운영 주체도 한국과 동일하게 **각 지방법원**이고, 통합 사이트 이름도 **BIT(不動産競売物件情報サイト)** — 한국의 `courtauction.go.kr`에 대응합니다.

법체계가 일본 민사집행법 → 한국 민사집행법으로 이어진 관계라 구조가 거의 같고, 본인 코드베이스를 일본판으로 확장하는 건 *전혀 다른 시스템 구축*이 아니라 *현지화(localization)* 수준의 작업으로 가능합니다.

## 한국 vs 일본 — 핵심 비교

| 항목 | 한국 | 일본 |
|---|---|---|
| 평가 주체 | 감정평가사 → 법원 | 부동산감정사 → 법원 |
| 평가액 명칭 | 감정가 | 売却基準価額(매각기준가액, 2005.4.1 도입) |
| 최저 입찰가 | 감정가 (1회차) | 買受可能価額(매수가능가액) = 매각기준가액의 80% |
| **1회차 시작가** | **감정가 100%** | **이미 감정가의 약 70~80% 수준에서 시작** |
| 유찰 시 가격 | 1회 유찰 시 자동 30% 하락 (서울 등 일부 20%) | **자동 차감 룰 없음**. 특별매각 후 미낙찰 시 법원이 평가를 다시 해서 가격을 정함 |
| 유찰 횟수 한도 | 사실상 없음 (7회 유찰 물건도 존재) | 대부분 3회 안에 처리, 3회 미낙찰 시 원칙적으로 절차 정지 |
| 입찰 보증금 | 최저가의 10% (특별 조건 시 20%) | 매각기준가액의 20%가 통상 |
| 결정적 자료 | 매각물건명세서 / 현황조사서 / 감정평가서 (3종) | 物件明細書 / 現況調査報告書 / 不動産評価書 — **三点セット(삼점세트)** |
| 사건번호 | `2024타경5532` | `令和○○年(ケ)○○号` (임의경매), `(ヌ)` (강제경매) |
| 통합 사이트 | courtauction.go.kr | bit.courts.go.jp |

## 가장 큰 차이: "유찰 → 가격 하락" 메커니즘

### 한국
- 1회차는 감정가 100%에서 시작
- 매회 자동으로 20~30% 깎임
- 7회 유찰되면 감정가의 18% 같은 극단치까지 떨어짐
- 예: 통영 `2024타경22056` — 7회 유찰 ▼82%, 감정가 2.5억 → 최저가 4,370만원

### 일본
- 1회차부터 이미 감정가 대비 약 30% 디스카운트된 *매각기준가액*에서 시작
- 그 80%(매수가능가액, 실질 시장가 약 56%)까지 입찰 가능
- 유찰돼도 **자동으로 추가 차감되지 않음**
- 미낙찰 → 특별매각(선착순) → 그래도 안 팔리면 법원이 평가를 다시 해서 가격을 새로 결정
- 3회 정도까지가 보통, 그 이후엔 절차 자체가 멈춤

거칠게 비유:
- **한국**: *"안 팔리면 자동으로 또 깎고, 또 깎고…"*
- **일본**: *"세 번 안에 처리, 안 되면 그냥 멈춤"*

## 비슷한 점

- 임의경매(担保不動産競売) / 강제경매(強制競売) 구분
- 점유자·임차인 권리 분석 필요성
- 잉여주의 (무잉여 시 경매 불성립)
- 보증금 미납 시 몰수
- 대금 일괄납부 원칙
- 인도명령·인도소송 같은 명도 절차 존재

## 일본만의 특징

### 1. BIT의 개인정보 마스킹
BIT에는 점유자·임차인 등의 고유명사가 검게 칠해진(黒塗り) 상태로 공개됩니다. 실제 입찰 시에는 **법원에 직접 가서 원본을 열람**해야 합니다. 한국 courtauction이 처음부터 이름까지 공개되는 것과 대비됩니다.

### 2. 1만 엔 함정 물건
매각기준가액이 `10,000円`으로 표기된 맨션이 BIT에 종종 등장합니다. 관리비·수선적립금 체납액이 커서 실질 평가가 마이너스인 경우입니다. 1만 엔에 낙찰돼도 **수백만 엔의 체납을 떠안아야** 합니다. 한국으로 치면 *유치권 있는 빌라*와 비슷한 함정.

이런 물건은 보증금 2,000엔만 내고 입찰한 뒤 낙찰 후 체납액을 보고 보증금을 그냥 포기하는 사례가 반복돼서 매물이 영원히 안 빠지는 현상이 있습니다.

### 3. 도시부의 가격 역전
도쿄·요코하마 같은 인기 지역에서는 실제 낙찰가가 매각기준가액의 **1.5배 이상**으로 가는 경우가 흔합니다. "싸게 산다"는 환상은 일본도 한국도 마찬가지로 깨집니다.

### 4. 절차의 단계
```
期間入札 (기간입찰) → 미낙찰 → 特別売却 (특별매각, 선착순) → 미낙찰 → 평가 재조정 → 재입찰
```
*특별매각*이라는 단계는 한국에는 없는 개념입니다.

## 본 프로젝트 관점의 시사점

### 데이터 모델 변경점

1. **가격 필드 이중화 필요**
   - 한국: 감정가 / 최저가 (2개)
   - 일본: 鑑定評価額 / 売却基準価額 / 買受可能価額 (3개)
   - 컬럼: `appraisal_value`, `sale_standard_price`, `purchase_possible_price`

2. **유찰 자동 차감 표시 무의미**
   - 한국 사이트의 `▼51%` 같은 할인 배지는 일본판에서는 다른 의미로 재해석 필요
   - 대신 *"평가 변경 이력"* 표시가 더 중요 — `valuation_history` 테이블 신설

3. **특별매각 단계 추가**
   - 사건진행상태에 `special_sale_period` 상태값 추가 필요
   - 기간입찰 / 특별매각 / 평가 재조정 대기 / 재입찰 — 4단계 상태머신

4. **사건번호 포맷**
   - 일본: `令和[연호]年([事件符号])第[번호]号` 패턴
   - 정규식: `/令和(\d+)年\((ケ|ヌ)\)第(\d+)号/`

### UX 차별화 포인트

5. **삼점세트 PDF 직접 노출이 한국판보다 더 핵심**
   - 일본 BIT는 개인정보 마스킹 때문에 법원 직접 열람을 강제하는 구조
   - 보조 도구가 **PDF 미러링·OCR·요약**을 해주면 한국판보다 더 큰 가치
   - LLM으로 삼점세트 요약·위험요소 추출하는 기능이 한국판보다 효과적

6. **1만 엔 함정 물건 경고 배지**
   - 일본 특화 기능
   - 매각기준가액이 비정상적으로 낮은 물건(예: < 100,000엔) 자동 검출
   - 관리비 체납 가능성 경고

7. **인구 감소 지역 표시**
   - 일본은 빈집(空き家) 문제가 심각 — 전체 주택의 13.8%
   - 지방 자치체별 인구 추이·빈집률 데이터 결합하면 가치 추가
   - 한국에는 없는 일본 특화 분석

8. **외국인 투자자 모드**
   - 엔저 영향으로 외국인(특히 한국·중국·싱가포르) 일본 부동산 관심 증가
   - 한국어 UI를 그대로 살리면 한국 거주 일본 부동산 투자자 타겟 가능

## 정찰 체크리스트

본인 한국판처럼 BIT를 분석하려면 다음을 정찰해야 합니다.

- [ ] BIT의 검색 API 엔드포인트 (한국과 달리 WebSquare RIA가 아닐 가능성)
- [ ] 인증 / 세션 / Rate limit 정책
- [ ] 페이지당 결과 수 제한
- [ ] 삼점세트 PDF 다운로드 URL 패턴
- [ ] 좌표 체계 (한국 KATEC와 다른 일본 측지계: JGD2000/JGD2011)
- [ ] 사건번호 → docid 매핑 규칙
- [ ] 매각결과(낙찰가) 공개 범위와 보존 기간

> ⚠️ 한국 `courtauction.go.kr`는 IP 차단 시 90초 대기 후 재시도하는 패턴이었음. BIT도 비슷한 방어 로직이 있을 가능성이 큼.

## 단계별 로드맵

1. **BIT 정찰** — 검색·상세 API의 reverse engineering. `docs/bit_api_recon.md` 작성

3. **스키마 설계** — 한국판 `properties` 테이블을 일본판으로 확장 가능하게 일반화 (또는 별도 `jp_properties` 분리)
